Functional & Symbolic Programming

The great and growing interest in functional and symbolic programming is due to the profound benefits they offer to the clarity, concision, reusability, and refactorability of code. Properly understood and deployed, these styles of programming can help more ambitious projects get done better, faster. They free programmers to focus on the essence of the problem at hand, rather than the invisible web of implicit interactions that characterizes imperative code. They therefore allow more mental (and often computational) resources to be devoted to deep problems and big ideas. These idioms are uniquely well suited to creative coding, and much of my work is concerned with bringing them to real-world, commercial creative coding contexts.

  • Arcadia is a full port of Clojure and the Clojure compiler to the Unity gaming platform and the CLR virtual machine, with all the facilities necessary to develop modern commerical-quality games in this advanced functional Lisp.
  • Cloze is a symbolic templating system designed to impose computable order on complicated data and markup.
  • The integration of Kovas Boguta's Gamma library into Arcadia brings powerful symbolic programming techniques to bear on the generation of shaders for Unity projects.
  • Live functional programming is a natural fit for virtual reality, enabling developers to program a running universe from within.
  • Hydrate defines efficient conversions between lucid functional data and imperative objects, giving programmers the best of both worlds.

Hydrate

Hydrate is an experimental system defining efficient bidirectional bindings between imperative and functional data.

In particular, Hydrate defines a bidirectional representation of the Unity scene graph, for the first time enabling its manipulation as functional data.

This transformation offers straightforward solutions to a number of hard problems in Unity, such as scene merging; diffing; well-defined serialization; version control; and snapshotting, while removing the need to use ungainly aspects of Unity such as prefabs.

Cloze

Cloze is a symbolic templating and rewrite system that imposes a simple and elegant associative structure on otherwise complicated data formats.

It is often difficult to express complex compositions of data using normal functional composition. Parameters for functions deep in the tree need to be propogated through all higher function calls, yielding a tangle of padded function arguments and constant pressure to tediously refactor. Once the structure has been constructed it is effectively opaque to simple interrogation and subsequent transformation, important locations obscured by the complexity of the surrounding structure.

This complexity is a wasteful, inelegant distraction from the meaning of the DSL. One could respond by observing a tight coding discipline that constrains generated structures in the DSL to those that can be easily parametrized by functional composition, but that merely moves the goalpost.

Rewrite systems decouple the shape of code from the shape of the data it processes.

Associative data structures provide clear self-documenting paths into potentially nested data under a declarative interface.

Cloze combines the advantages of rewrite systems and associative data structures to abstract the emergent parameterization of complex data into relatively simple associative paths, fully compatible with Clojure's standard library of data processing functions.
